STEP 2: AIから修正コードを受け取る

編集 2/9

送信すると数秒〜10秒ほどで、Claude が以下のような回答を返してくれます。

Claude の回答:
承知しました。以下が変更内容です。

1) ファイル: index.html
2) 検索キーワード（Ctrl+F）: ¥50,000

4) 注意点: 英語版 en/index.html にも同じ価格が記載されている場合は両方変更してください。

📥 ここで「コピー」するもの

1「after（変更後）」のコードブロック右上の📋アイコンをクリック

Claude のコードブロックには右上に小さな「コピー」ボタンがあります。📋 Copy をクリックするとクリップボードにコピーされます。

2「検索キーワード」もメモしておく

この例なら ¥50,000。STEP 5 で GitHub内検索に使います。

before（変更前）のコードは絶対に貼り付けないでください。 必ず「after（緑のブロック）」をコピーすること。混同を防ぐため、AIには毎回「before/afterを分けてください」と頼みましょう。

📌 AIに聞き返してOK

「もっと具体的に説明して」「該当箇所のスクリーンショット風に書いて」「英語版も変更が必要？」など、納得するまで何度でも質問できます。1往復で完璧な答えが返らなくても全然OK。

不安な時は「この変更でサイトが壊れる可能性はありますか？」と聞いてみてください。Claudeがリスクを評価してくれます。

STEP 5: 該当箇所を検索する

編集 5/9

index.html は約1500行あります。手でスクロールして探すのは大変。キーボードショートカットで一発検索します。

1エディタ画面の中で Ctrl + F（Macは ⌘ + F）を押す

画面右上に検索バーが出現します。

✏️ Editing index.html
🔍 1 of 1 ここに入力
845 <div class="campaign">
846 <h3>限定キャンペーン</h3>
847 <span class="price">¥50,000</span>
848 </div>
2STEP 2 で AI からもらった検索キーワード（例: ¥50,000）を入力

該当箇所が 黄色〜オレンジ色のハイライトで表示されます。複数該当する場合は、検索バーの「1 of 3」のような表示の矢印で次の箇所に移動できます。

3Esc キーで検索バーを閉じる

該当行が画面の中央付近に表示された状態になっているはずです。

🎯 検索のコツ

  • 記号は省略OK → 「<span>」より「price」のようなクラス名

複数該当して「どれが正解？」と迷ったら、編集はせずに Claude にスクショを貼って質問してください。間違った箇所を編集すると別の場所が壊れます。

STEP 6: コードを貼り付ける

編集 6/9

いよいよ核心。STEP 2 でコピーした「after（変更後）コード」を、検索でハイライトされた行に上書きします。

1置き換える行を選択（マウスでドラッグ）

該当行の左端から右端までマウスでドラッグして選択。または、行をトリプルクリック（3回連続クリック）すると、その行全体が選択されます。

847 <span class="price">¥50,000</span> ← 選択
↓ Ctrl+V で貼り付け ↓
847 <span class="price">¥45,000</span> ✓ 上書き完了
2Ctrl + V（Macは ⌘ + V）で貼り付け

選択した行が、AIからコピーした「after」のコードで上書きされます。

3貼り付け後の見た目を目視確認

「¥50,000」だった箇所が「¥45,000」になっていればOK。前後の <span> タグは残っていますか？

✅ 貼り付け後のセルフチェック

確認項目OK例NG例
タグの数<span>〜</span> が1セット<span><span>〜</span>（二重）
記号の対応クォート " がペア片方だけ消えている
インデント前後の行とスペース数が揃ってる左端にくっついてる
「貼り付けたら大量の行が変わってしまった！」時は要注意。 何も保存せず、Ctrl+Z（取り消し）を連打して元に戻してください。それでもダメなら、ブラウザのタブを「保存せず閉じる」だけで全部リセットされます。
この時点ではまだ 本番サイトには何の影響もありません。STEP 8 の「Commit changes」を押すまでは下書き状態です。安心して試行錯誤してOK。

STEP 7: プレビューで確認

編集 7/9

保存する前に、「Preview」タブで「どこをどう変えたか」を視覚的に確認できます。これ、めちゃくちゃ重要です。

1画面上部の「Preview」タブをクリック
Edit Preview これをクリック Commit changes...
2「変更前 → 変更後」が緑/赤でハイライトされる
847 − <span class="price">¥50,000</span>
847 + <span class="price">¥45,000</span>

−（赤） 削除される行
＋（緑） 追加される行

🎯 プレビューで必ず確認する3点

  1. 変更したい箇所だけが赤緑になっているか — 関係ない箇所が変わっていたら誤編集
  2. 赤と緑が同じ行数か — 例: 赤1行・緑1行ならOK。赤1行・緑3行は何かズレてる
  3. 意図した変更内容になっているか — 「¥45,000」になっているか目視確認

❌ プレビューでおかしい時の対処

症状対処
大量の行が赤緑になっている「Edit」タブに戻って Ctrl+Z を連打 → ブラウザのタブを閉じる
意図した箇所と違う同上。一度全部リセットして STEP 5（検索）からやり直し
HTMLタグが壊れて見える同上。Claude にスクショを貼って「これ大丈夫？」と確認
プレビューで「これでOK！」と確信できたら、次の STEP 8（Commit）に進みます。不安なら何度でも Claude に質問してOK。確信できるまで Commit しないこと。

🆘 困った時：前の状態に戻す

トラブル対処

🔄 万が一サイトが壊れた時の「1分で元に戻す」方法

1GitHub リポジトリのトップで「Commits」をクリック

URL: github.com/koki-187/scalp-ink-lp/commits/master

2「正常だった頃」のコミットを探す

過去のコミット履歴が新しい順に並んでいます。「キャンペーン価格を変更」のひとつ前など、正常だった状態を探します。

3そのコミットの「<> Browse files」をクリック

当時のファイル一覧が表示されます。

4最新の壊れたコミットを「Revert」する

または、AIに「最新のコミットを取り消したい」と相談すると手順を案内してくれます。

慌てない！ GitHubは すべての変更履歴を永久に保管しています。「やらかした」と思っても、必ず復元できます。一人で抱え込まず、Claude → 開発者の順で相談してください。

📞 連絡先・サポート優先順位

優先順位対象連絡方法
1Claude（AI）に相談claude.ai でスクショを貼って質問
2このガイドPDFを再読該当ステップを見直す
3開発担当者（緊急時のみ）Bitwardenに記載の連絡先

🚨 開発担当者に連絡すべきレベル

  • サイトが完全に表示されない（404 / 500 エラー）
  • 同じエラーが何度もループする
  • GitHubにログインできなくなった
  • Cloudflare Pages から「Build failed」のメールが届いた

逆に「文字の色をもう少し濃くしたい」「画像を差し替えたい」「新セクションを足したい」レベルは、すべて Claude と GitHub で完結します。臆せずチャレンジしてください。

📚 よくある編集レシピ集

付録

毎回ゼロから AI に説明するのは面倒なので、「コピペで使える依頼文テンプレ」 を用途別に用意しました。

🔧 レシピ1: テキスト・価格の変更

index.html の「【変更前のテキスト】」を「【変更後のテキスト】」に変更したい。 before/after コードと、Ctrl+F検索キーワードを教えて。

🔧 レシピ2: 営業時間の変更

index.html の営業時間表示を「平日10:00-20:00 / 土日祝9:00-19:00」に変更したい。 該当箇所のbefore/afterコードを出して。

🔧 レシピ3: 電話番号の変更

index.html と en/index.html の両方で、電話番号を【新しい番号】に変更したい。 tel: リンクと表示テキストの両方を更新するコードを教えて。

🔧 レシピ4: 画像の差し替え
