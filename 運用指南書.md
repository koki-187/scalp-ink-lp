## 3. 日常運用 — 予約受付フロー

### 3-1. 予約が入った時の流れ

1. **予約者がサイトの予約フォームから送信**
2. **以下が自動実行:**
   - Google Sheets「予約管理」シートに新規行が追加
   - `cimascalp.lp@gmail.com` 宛に件名「【新規予約】○○様」でメール通知
   - 予約者に自動返信メール「予約リクエスト受付けました」

### 3-2. 担当者の対応手順

**📬 メール通知を受信したら:**

1. [予約管理シート](https://docs.google.com/spreadsheets/d/1Zm75MWcwX-puHbfFoLLycsBh8hulFvwuKgKROypsb4I/edit) を開く
2. 新規追加された行の内容を確認
   - 受信日時 / お名前 / 電話番号 / メール / 希望日時 / メニュー / メッセージ
3. **24時間以内** に以下のいずれかで連絡:
   - メールで直接返信
   - Instagram DM で連絡
   - LINE公式アカウント宛に友達追加依頼
4. 対応完了後、シートの「ステータス」列を `未対応` → `連絡済` / `予約確定` / `キャンセル` 等に更新
5. 「対応メモ」列に日時・内容を記録（任意）

### 3-3. 東京・沖縄拠点のスケジューリング

現在は **拠点スケジュール未確定**のため、個別対応が基本:
- 予約者から希望日時を確認
- 東京/沖縄のどちらが便利かヒアリング
- CIMA様のスケジュールと照合して確定日を決定

### 3-4. スプレッドシート運用ルール

- **行を物理削除しない** — 「ステータス」列で管理
- キャンセル時は `キャンセル` ステータスに更新し、対応メモに理由を記録
- 月次で過去3ヶ月分を別シート「予約履歴」にアーカイブ推奨

## 4. 紹介特典の管理運用

### 4-1. 紹介特典規定（tokushoho.html 記載）

| 対象 | 特典 |
|---|---|
| **紹介者**（既存のお客様） | 被紹介者の施術＋支払い完了後に 5,000発プレゼント（有効期限12ヶ月） |
| **被紹介者**（新規のお客様） | 施術時に 5,000発加算（初回割引との併用不可） |

### 4-2. 運用手順（推奨フロー）

**🎁 新規予約で紹介コードを受領したら:**

1. [予約管理シート](https://docs.google.com/spreadsheets/d/1Zm75MWcwX-puHbfFoLLycsBh8hulFvwuKgKROypsb4I/edit) の「紹介特典管理」タブを作成（未作成の場合）

  ヘッダー:
  ```
  | 紹介日 | 紹介者 | 被紹介者 | 残ドット数 | 有効期限 | 使用状況 |
  ```

2. 行を追加:
   - 紹介日: 被紹介者の施術完了日
   - 紹介者: 既存のお客様の氏名（LINE名などで特定）
   - 被紹介者: 新規のお客様の氏名
   - 残ドット数: 5,000
   - 有効期限: 紹介日 + 12ヶ月（例: 2026-04-23 → 2027-04-23）
   - 使用状況: 未使用

3. **Google カレンダーに失効リマインダーを登録:**
   - タイトル: 「【SCALP INK】○○様（紹介者）特典失効予定」
   - 日時: 有効期限の7日前
   - 通知: 1週間前、1日前

4. **紹介者の LINE/Instagram に通知:**
   - 「ご紹介ありがとうございました。5,000発の紹介特典を進呈いたします（有効期限: YYYY-MM-DD）」

### 4-3. 紹介特典使用時

- 紹介者が施術予約時に特典適用を希望
- シートの残ドット数から使用分を減算
- 使用状況を `使用済` に更新
- 全額使用した場合は行を「紹介特典履歴」タブにアーカイブ

## 5. AIシミュレーター運用

### 5-1. 機能概要

ユーザーが頭部写真をアップロード/撮影 → Gemini API が解析 → SMP施術シミュレーション画像を生成

### 5-2. 運用上の注意

- **無料枠:** Gemini API は無料枠で運用中（Cloudflare Worker 経由）
- **レート制限:** 429エラー発生時は2秒バックオフで自動リトライ
- **ユーザーデータ:** 画像はサーバーに保存されず、解析後は破棄

### 5-3. ユーザーからの問い合わせ対応

| 問い合わせ | 対応 |
|---|---|
| 「シミュレーターが動かない」 | 1. ブラウザのハード再読込（Ctrl+F5） 2. 別ブラウザで試す（Chrome/Safari） 3. 画像サイズが大きすぎないか確認 |
| 「結果画像が不自然」 | AI自動検出モード or 黄色ペンで範囲指定モードを試すよう案内 |
| 「コードエラーが出る」 | GitHub Issues に報告 or 開発者に連絡 |

### 5-4. AI解析レポートの読み方

レポートは5段階で構成:
1. **STEP 1:** 状態タイプ特定（薄毛全体型/ヘアライン後退型/傷跡/脱毛パッチ/複合型）
2. **STEP 2:** 詳細状態分析（進行度・面積・肌色トーン）
3. **STEP 3:** SMP施術プラン（推奨ドット数・セッション数・時間）
4. **STEP 4:** 施術後のビジュアルイメージ（密度感・自然さテーブル）
5. **STEP 5:** 施術後のアフターケア推奨事項

## 8. コンテンツ更新方法

### 8-1. 写真・画像の差し替え

**Before/After 写真の更新:**
1. 新しい画像を `assets/ba/` ディレクトリに同名（例: `hairline-1-after.jpg`）で配置
2. 画像サイズは **900×900px、JPEG、150KB前後** を推奨
3. GitHub に push → 自動デプロイ

**OGP画像（SNSシェア用）:**
- パス: `assets/ogp.png`
- サイズ: 1200×630px
- 同パスで上書き

### 8-2. テキストの更新

**料金変更の場合の修正箇所（全て同期が必要）:**
1. `index.html` の料金セクション
2. `en/index.html` の料金セクション
3. `tokushoho.html` の料金体系
4. `en/tokushoho.html` の料金体系
5. JSON-LD の `makesOffer` 配列（両ファイル）
6. Apps Script の予約メールテンプレート（必要に応じて）

### 8-3. 営業時間変更

1. `index.html` の JSON-LD `openingHoursSpecification`
2. `en/index.html` の同箇所
3. LINE公式アカウントの営業時間設定
4. Search Console のビジネスプロフィール（あれば）

## 9. トラブルシューティング

### 9-1. 予約通知が届かない

1. `cimascalp.lp@gmail.com` の**迷惑メールフォルダ**を確認
2. [予約管理シート](https://docs.google.com/spreadsheets/d/1Zm75MWcwX-puHbfFoLLycsBh8hulFvwuKgKROypsb4I/edit) に行が追加されているか確認
3. 行はあるがメールが来ない場合:
   - [Apps Script エディタ](https://script.google.com/home) で該当プロジェクトを開く
   - 左メニュー「実行数」で直近のエラーログを確認

### 9-2. AIシミュレーターで 429 エラー

- 短時間に多数のリクエストがあった場合の Gemini レート制限
- **対応:** 1〜2分待ってから再試行
- 頻発する場合は Cloudflare Worker のログを確認

### 9-3. サイトが表示されない

1. https://www.cloudflarestatus.com/ で Cloudflare 全体の障害を確認
2. [GitHub リポジトリ](https://github.com/koki-187/scalp-ink-lp) で最近のコミットを確認
3. Cloudflare Pages ダッシュボードでデプロイ状況を確認

### 9-4. LINE の応答が止まっている

- [LINE Official Account Manager](https://manager.line.biz/account/@045ovoxr) → 自動応答 → 応答メッセージ
- 各テンプレートの「利用停止」がOFFになっているか確認
